# EU Password Security Policy

## Purpose
This document defines the password requirements for European Union business systems. It is intended to improve account security for employee login, application access, and remote work scenarios. The policy applies to corporate accounts used for email, HR portals, collaboration tools, and VPN access.

## Scope
The requirements in this version applied to:
- EU employees with corporate user accounts
- EU contractors with managed directory accounts
- Internal and approved cloud applications integrated with enterprise authentication

## Password Requirements
Users were required to create passwords that met all of the following:
- Minimum length of 12 characters
- At least one uppercase letter, one lowercase letter, one number, and one special character
- No reuse of the last 8 passwords
- No obvious personal data such as first name, birth year, or employee ID

Passphrases were permitted as long as they satisfied the same length and complexity requirements.

## Password Change Policy
To reduce long-term exposure risk, passwords were subject to periodic rotation:
- Password change required every 90 days
- Forced reset at expiration if not changed proactively
- Helpdesk-assisted reset allowed after identity verification

## Multifactor Authentication
Multifactor authentication (MFA) was supported across key platforms:
- MFA was recommended for remote access
- MFA was mandatory only for selected high-risk systems
- Some legacy internal tools were still password-only

## Account Recovery and Temporary Access
When a user forgot their password or account was locked:
- Service Desk issued a temporary password after verification
- Temporary password expired within 24 hours
- User had to set a new compliant password at next login

## Compliance and Exceptions
Managers and application owners were expected to ensure compliance. Exceptions for legacy systems could be approved by IAM and recorded with a remediation date.
