# EU VPN Access Procedure

## Purpose
This procedure explains how EU personnel request, activate, and use corporate VPN access for secure remote connectivity. It also defines authentication and password-related controls that apply during remote login.

## Scope
This procedure applies to:
- EU users connecting from home or travel networks
- Managed corporate devices authorized for remote access
- Approved contractors with assigned EU directory accounts

## Prerequisites
Before VPN access can be enabled, users must:
- Have an active corporate identity account
- Complete endpoint compliance checks
- Enroll in approved multifactor authentication methods
- Confirm account credentials comply with the active EU password policy

## Access Request Steps
1. Submit VPN access request in Service Portal with business justification.
2. Manager approves request and confirms remote-work requirement.
3. Network Operations validates device posture and account status.
4. IAM verifies authentication enrollment and account eligibility.
5. User receives activation notice and setup instructions.

## First-Time VPN Setup
After approval, users should:
- Install the approved VPN client from internal software catalog
- Sign in with corporate username and password
- Complete MFA challenge during initial authentication
- Select EU gateway profile when prompted
- Verify successful connection to internal resources

## Authentication and Password Controls
Users must maintain secure login behavior:
- Password sharing is prohibited
- Temporary passwords must be changed immediately at next login
- Repeated failed authentication attempts may lock VPN access
- MFA is mandatory for all VPN sessions
- Users must follow the active EU password policy standard for credential requirements

For password complexity, minimum length, and reset triggers, refer to the active policy document managed by IAM. This procedure does not replace the policy.

## Support and Incident Handling
If login or authentication fails:
- Open a high-priority Service Desk ticket
- Include error code, timestamp, and network context
- Service Desk may issue guided password reset steps after verification
- Security team may enforce temporary suspension for suspected compromise

## Compliance Notes
VPN logs are monitored for anomalous access. Unauthorized credential use, MFA bypass attempts, or policy violations may result in access revocation.
