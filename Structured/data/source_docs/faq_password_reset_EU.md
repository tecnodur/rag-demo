# EU Password Reset FAQ

## Overview
This FAQ answers common questions about password reset, account lockout, temporary credentials, and authentication support for EU users. It is intended for operational guidance and does not replace the official password policy.

## 1. When should I reset my password?
Reset your password when:
- You forgot your current password
- You suspect account compromise
- Service Desk or Security instructs a reset
- You receive a verified security notification requiring credential update

Routine reset frequency is defined by the active policy, not by this FAQ.

## 2. How do I reset my password?
Use the self-service portal:
1. Open the internal account recovery page.
2. Enter your corporate username.
3. Complete identity verification challenge.
4. Set a new password or passphrase that meets policy requirements.
5. Re-authenticate in key applications and VPN client.

If self-service is unavailable, contact Service Desk for assisted reset.

## 3. What happens if my account is locked?
Accounts may lock after repeated failed login attempts. To recover:
- Wait for automatic unlock window if shown
- Use approved unlock flow in recovery portal
- Contact Service Desk if lockout persists
- Complete verification before manual unlock is applied

## 4. What is a temporary password?
A temporary password is a short-lived credential provided after identity verification. Rules:
- One-time use only
- Must be changed immediately at next login
- Cannot be reused for future resets
- May require immediate MFA revalidation

## 5. Do I still need MFA after a password reset?
Yes. Password reset does not remove MFA requirements for remote access or sensitive applications. Users may need to re-enroll factors if device trust changed.

## 6. Where can I find the official EU password rules?
The authoritative source is the active EU password policy maintained by IAM. It defines minimum password length, passphrase guidance, authentication requirements, and reset triggers.

## 7. Who do I contact for urgent help?
For urgent access issues:
- Contact Service Desk by phone and mark ticket as "EU Login Critical"
- Notify your manager if business-critical tasks are blocked
- Escalate to Identity Support for repeated authentication failures
